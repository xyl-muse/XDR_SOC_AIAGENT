# 系统架构文档

## 概述

XDR_SOC_AIAGENT是一个将XDR安全运营中心与企业智能体平台集成的安全运营自动化系统。通过API对接实现安全事件的全生命周期管理，从日志采集、告警分析、事件研判到工单处置的完整闭环。

## 核心模块

### 1. AGENT函数API层

企业智能体平台调用的核心接口层，提供以下能力：

| 接口 | 功能 | 方法 |
|------|------|------|
| `/incidents/:uuid/proof` | 获取事件举证信息 | GET |
| `/incidents/:uuid/entities/process` | 获取事件进程实体 | GET |
| `/incidents/status` | 修改事件处置状态 | POST |
| `/callback/incident` | 事件分析结果回调 | POST |

### 2. 溯源专家模块

负责安全事件的溯源分析：

```
输入: hostIp (IP地址)
     ↓
IP格式校验
     ↓
资产归属查询 ──→ 大区、分公司
     ↓
资产类型查询 ──→ 办公系统/业务系统/办公PC
     ↓
输出: JSON格式资产信息
```

### 3. 处置专家模块

对接工单系统，实现自动化处置：

```
安全事件
    ↓
事件等级判定
    ↓
┌───┴───┐
│ ITSM  │ ←── 日常安全事件
└───┬───┘
    │ 创建工单 → 分派 → 处置 → 归档
    ↓
┌───┴───┐
│  MPT  │ ←── 重大安全事件
└───┬───┘
    │ 创建专项工单 → 应急响应 → 复盘
    ↓
处置完成
```

### 4. 研判专家模块

安全事件智能研判：

```
事件数据输入
    ↓
威胁情报关联
    ↓
攻击链分析
    ↓
影响评估
    ↓
处置建议生成
    ↓
研判报告输出
```

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐

│  采集    │    │  生成    │    │  聚合    │    │  分析    │    │  执行    │

     │              │              │              │              │
     ▼              ▼              ▼              ▼              ▼
  EDR日志       规则匹配      关联分析      智能研判      工单创建
  NDR日志       威胁检测      攻击链构建    报告生成      状态更新
```

### 数据格式规范

项目支持以下日志格式：

| 日志类型 | 数据源 | 关键字段 |
|---------|--------|---------|
| 安全告警 | NDR/EDR/AF | alertType, threatType, riskTag |
| 安全事件 | XDR聚合 | incidentSeverity, attackStory |
| 端点日志 | EDR | virusName, filePath, fileMd5 |
| 网络日志 | NDR/AF | httpUri, srcIp, dstIp, ruleName |
| 终端行为 | EDR | eventId, pName, dnsQName |
| DNS日志 | DNS审计 | queries, answers, rCode |

## 认证机制

### HMAC-SHA256签名流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        签名生成流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 解码auth_code获取AK/SK                                      │
│     auth_code ──▶ hex解码 ──▶ AES-CBC解密 ──▶ AK, SK           │
│                                                                 │
│  2. 构造规范请求字符串 (CanonicalRequest)                        │
│     Method + URL + QueryString + Headers + Payload              │
│                                                                 │
│  3. 计算签名                                                    │
│     SignDate = ISO8601格式时间                                  │
│     HashedCanonicalRequest = SHA256(CanonicalRequest)          │
│     StringToSign = "HMAC-SHA256\n{Date}\n{Hash}"               │
│     Signature = HMAC-SHA256(SK, StringToSign)                  │
│                                                                 │
│  4. 组装Authorization头                                         │
│     Authorization: algorithm=HMAC-SHA256, Access={AK},          │
│                    SignedHeaders={headers}, Signature={sig}     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 代码示例

```python
# 初始化签名对象
signature = Signature(auth_code="YOUR_AUTH_CODE")

# 构造请求
req = requests.Request("GET", api_url, headers=headers)

# 签名
signature.signature(req=req)

# 发送请求
response = requests.Session().send(req.prepare())
```

## 扩展性

### 新增API接口

1. 在`AGENT函数API/`目录创建新的Python文件
2. 实现`userFunction(inputs)`入口函数
3. 配置签名认证逻辑
4. 在智能体平台注册新接口

### 新增外部系统集成

1. 在对应专家目录创建集成脚本
2. 实现认证和API调用逻辑
3. 配置Webhook回调

## 监控与日志

- **API调用日志**：记录所有API请求和响应
- **执行日志**：记录智能体执行过程
- **错误日志**：记录异常和错误信息

## 安全加固

1. **传输加密**：全链路HTTPS
2. **认证加密**：HMAC-SHA256签名
3. **凭据管理**：环境变量注入
4. **访问控制**：IP白名单限制
